== 3. Background

=== 3.1. ReAct Pattern

The ReAct (Reason + Act) pattern interleaves:

. *THOUGHT* -- Analyze what needs to be done
. *ACTION* -- Choose and invoke a tool
. *OBSERVATION* -- Review the tool's result
. Repeat until a final answer emerges

=== 3.2. Stateful Memory

Agents require memory to track progress across turns.
We use Redis-backed `MessageWindowChatMemory` for session persistence.

=== 3.3. Multi-Agent Orchestration

Complex tasks benefit from specialized agents (support, documentation, product) coordinated by a routing agent.
