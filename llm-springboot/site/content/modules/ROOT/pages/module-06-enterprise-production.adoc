=== What You Will Build

A production-ready deployment of the Enterprise Knowledge Assistant on OpenShift with observability, caching, and evaluation benchmarks.

== 3. Background

=== 3.1. LLM Evaluation (Evals)

Automated benchmarks measure accuracy, relevance, and faithfulness using a "golden dataset" of known question/answer pairs.

=== 3.2. Observability

Distributed tracing (OpenTelemetry) and metrics (Micrometer/Prometheus) give visibility into latency, token costs, and decision pathways.

=== 3.3. Optimization

* *Semantic caching* reduces redundant LLM calls.
* *Token optimization* controls context window size and prompt compression.

== 4. Hands-On Lab

=== Lab 6.1: LLM Evaluation Framework

==== Objective

Create a golden evaluation dataset and run benchmarks.

==== Steps

. Create `EvaluationService` that loads eval cases and measures accuracy, relevance, and faithfulness.
. Prepare a golden dataset in `src/main/resources/data/eval-golden-set.json`.
. Run the evaluation and review the report.

=== Lab 6.2: Distributed Tracing

==== Objective

Implement distributed tracing across the RAG pipeline.

==== Steps

. Configure OpenTelemetry with `TracingConfig`.
. Add `@Traced` annotations to key service methods.
. Create `MetricsCollector` for query counts, response times, and token usage.
. Start Prometheus and Grafana:
+
[source,bash]
----
docker compose up -d prometheus grafana
----

. Verify traces and metrics appear in Grafana.

=== Lab 6.3: Semantic Caching

==== Objective

Implement semantic caching to reduce LLM calls.

==== Steps

. Create `CachingService` with both exact-match and semantic cache.
. Configure a similarity threshold for cache hits.
. Measure the cache hit rate and latency improvement.

=== Lab 6.4: Token Optimization

==== Objective

Optimize token usage while maintaining quality.

==== Steps

. Create `TokenOptimizer` that fits context within a token budget.
. Add prompt compression to reduce filler words.
. Compare output quality before and after optimization.

=== Lab 6.5: Deploy to OpenShift

==== Objective

Deploy the application to OpenShift with monitoring.

==== Steps

. Build the container image:
+
[source,bash]
----
docker build -t techcorp-assistant:latest -f deployment/Dockerfile .
----

. Apply the deployment manifests:
+
[source,bash]
----
oc apply -f deployment/deployment.yaml
----

. Verify the application is running:
+
[source,bash]
----
oc get pods -l app=techcorp-assistant
oc get route techcorp-assistant-route
----

. Test the deployed endpoint.
