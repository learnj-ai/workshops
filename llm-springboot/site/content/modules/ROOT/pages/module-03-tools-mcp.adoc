== 3. Background

=== 3.1. The MCP Ecosystem

MCP defines a standard protocol for how AI models interact with external tools and data sources.
Key concepts:

* *Host* -- The application running the AI model
* *Client* -- The component that discovers and invokes tools
* *Server* -- The component that exposes tools and resources

=== 3.2. Langchain4J Tool Annotations

Langchain4J provides `@Tool` and `@P` annotations for declaring tools that the LLM can discover and invoke.

== 4. Hands-On Lab

=== Lab 3.1: Database Tool

==== Objective

Create a custom tool that queries the customer database.

==== Steps

. Start PostgreSQL:
+
[source,bash]
----
docker compose up -d postgres
----

. Create the `CustomerDataTool` class using `@Tool` annotations.
. Wire the tool into the chat model configuration.
. Test with natural language queries about customers.

=== Lab 3.2: External API Tool

==== Objective

Build a tool that fetches real-time data from a public API.

==== Steps

. Create a `WeatherTool` that calls an external weather API.
. Register it alongside the database tool.
. Test with queries that require the model to select the correct tool.

=== Lab 3.3: Tool Orchestration

==== Objective

Test tool selection with ambiguous queries.

==== Steps

. Create a `ToolOrchestrator` service that handles multi-turn tool invocations.
. Test with queries that chain multiple tools together.
